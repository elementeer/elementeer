# Elementify Free Product Surface

## Positioning

**The intelligent free lift and starting point for WordPress + Elementor**

Free is the public and mirror-safe product surface. It must feel like a real operational upgrade for Elementor users and a credible guided starting point for users who are not yet fully in the Elementor stack.

## Core Promise

Free makes WordPress + Elementor outcomes immediately more understandable, more guided, and more operationally useful without requiring premium assets or cloud services.

That includes:

- existing Elementor sites
- existing WordPress sites without Elementor
- new-site starts from an idea, offer, or early brand

## What Free Includes

### 1. Stack bootstrap and scenario entry

- intent wizard
- scenario-first runtime front door
- runtime wizard families:
  - new-site-lite
  - stack-bootstrap
  - optimization-lite
  - relaunch-lite
  - extension-lite
- limited guided transitions for the first safe Free paths
- stack bootstrap guidance
- baseline stack recommendation
- Hello Theme baseline guidance
- curated free addon profile recommendation where justified
- recommendation of the right next wizard path for guided, assisted, and technical users

Why this belongs in Free:

- Free must be a 360-degree starting point, not only an uplift for already-mature Elementor users.
- This is the front door for users starting from an idea or from WordPress without Elementor.

### 3. Brand setup and site foundation

- get global styles
- set global colors
- set global typography
- set site logo
- brand setup wizard

Why this belongs in Free:

- It creates visible lift quickly.
- It is one of the strongest “more than a demo” experiences.

### 4. Creator Light and simple assembly

- creator mode
- compose page from templates
- update page data
- get page data
- save page section as template
- save full page as template
- simple customization of existing Elementor structures

Why this belongs in Free:

- The launch strategy explicitly says Free may generate or assemble simple pages, sections, and widget structures.
- This is a key differentiator versus a pure audit or wizard-only story.

### 5. Local Elementor Library operations

- list, inspect, create, update, delete, rename, duplicate, and bulk rename templates
- read and update raw template data
- extract sections
- categorize and tag templates
- audit the current library

Why this belongs in Free:

- The local Elementor Library is the primary target system.
- Elementify should make the existing Elementor stack more useful, not replace it.

### 6. Agent-native setup and skill-facing entry

- list configured sites
- switch configured site
- skills catalog
- API-key-based operator setup
- guided vs technical onboarding posture

Why this belongs in Free:

- It is local operator convenience and part of the no-code-near / technical hybrid entry point.
- It is not the same as future Studio cloud reuse.

## Free Workflow Set

### Free workflow 1: Bootstrap from idea or incomplete stack

Move from:

- idea only
- brand without site
- existing WordPress without Elementor
- partial stack

to a sane supported baseline and a clear next path.

This includes:

- `new-site-lite` for idea-first and brand-first starts
- `stack-bootstrap` for WordPress-without-Elementor and partial-stack starts

### Free workflow 2: Assessment and recommendation loop

Understand the current site, store context, and derive prioritized next steps.

### Free workflow 3: Quickstart and guided onboarding

Move from “what kind of site is this?” to “what should I fix or improve first?”

### Free workflow 4: Brand setup and site foundation

Get to a usable logo, color, and typography baseline.

### Free workflow 5: Creator Light

Assemble simple structures from the current site and existing templates.

### Free workflow 6: Local library organization and customization

Turn the current Elementor Library into a better operational asset.

### Free workflow 7: Existing-site uplift

Support light and moderate paths for:

- clean-up
- refresh
- optimization
- extension
- reduction
- relaunch lite

Within the runtime layer:

- `clean_up` and `refresh` are carried by the `optimization-lite` family with scenario-specific wording
- `extension` is carried by `extension-lite`
- relaunch stays in `relaunch-lite` until the path clearly demands deeper Advanced workflow depth

## What Free Explicitly Does Not Include

- Theme Builder delivery workflows
- premium Elementify Library access
- change-queue governance workflows
- brand adaptation planning
- rebuild strategy planning and critique
- output critique and repair guidance
- AI image generation and premium media workflows
- cloud library promises
- cross-site delivery promises
- Studio orchestration promises

## Product Guardrails

- Free must remain public and mirror-safe.
- Free must build and test without private Advanced modules.
- Free may be operationally serious, including bootstrap guidance and delete/write behavior, as long as governance and scoped capabilities remain the safety layer.
- Free must never promise cloud-library or Studio semantics.
- Free may recommend curated skill profiles and free addon profiles, but must not collapse into extension inflation.
- The Free mirror must stay aligned with `registerFreeTools`, the public docs list, and the staged `free-tool-surface.json` snapshot produced from the runtime build.
- The public onboarding path lives in [docs/quickstart/free.md](/Users/andrelange/Documents/repositories/github/elementify-mcp/docs/quickstart/free.md).
- The public release checklist lives in [docs/release/free-public-release-checklist.md](/Users/andrelange/Documents/repositories/github/elementify-mcp/docs/release/free-public-release-checklist.md).

## Strategic Interpretation

Free is not “small Elementify”.

It is:

- the public reach layer
- the learning layer
- the waitlist and ecosystem layer
- the first real operational value layer
- the first useful stack-bootstrap layer
