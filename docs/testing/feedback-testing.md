# Elementify Pilot Feedback Log

## Purpose

This is the lightweight running log for real-world Elementify testing.

Use it during live pilots on:

- a real client or staging site
- the Elementify site itself

This log is intentionally simple.

Do not wait for perfect wording.
Capture friction, product gaps, wins, and ideas as they happen.

## How To Use

For each meaningful observation, add one entry.

Prefer short, direct notes over long narratives.

Use these types:

- `bug`
- `ux-friction`
- `copy`
- `missing-guidance`
- `missing-capability`
- `product-gap`
- `docs`
- `win`

Use these severities:

- `blocker`
- `high`
- `medium`
- `low`
