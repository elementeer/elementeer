# Elementify Advanced Product Surface

## Positioning

**The deeper workflow and productivity layer for ambitious WordPress + Elementor work**

Advanced is the first paid and private product layer. It must feel like a real productivity upgrade, not like Free plus a handful of locked extras.

## Core Promise

Advanced extends the same scenario families as Free with deeper workflow control, deeper creator power, stronger critique and adaptation, richer stack shaping, and the first visible Elementify Library experience.

## What Advanced Includes

### 1. Scenario-first Advanced front door

- private scenario routing for deep relaunch, migration, premium rollout, and critique / repair entry states
- explicit recommendation of the right Advanced workflow before execution begins
- scenario-aware action presets and operative handoffs instead of a flat private tool shelf

Why this belongs in Advanced:

- Advanced should feel like a deeper operating layer, not just a list of stronger tools.
- This is where the Productivity Upgrade feeling starts to become visible.

### 2. Deep scenario execution

- deep relaunch paths
- deeper migration support
- deeper optimization and extension support
- richer wizard depth for guided, assisted, and technical operators

Why this belongs in Advanced:

- Advanced should deepen the same scenario universe rather than feel like a disconnected extras tier.
- This is where Advanced becomes a real productivity upgrade.

### 3. Theme Builder delivery workflows

- Theme Builder wizard
- advanced creator mode for premium rollout, structural rollout, and critique loops
- deeper structural workflows for header, footer, archive, single, and related site-wide templates

Why this belongs in Advanced:

- It changes site-wide structural behavior.
- It is the most natural line between creator-light and deeper workflow control.

### 4. Deeper creation and premium media support

- stock image search
- stock image sideloading
- AI image generation

Why this belongs in Advanced:

- It increases creator power and guided generation depth.
- It improves workflow speed and output quality without being necessary to fulfill the Free promise.

### 5. Governed operations and higher-trust workflows

- queue change
- list change queue
- review change
- apply change

Why this belongs in Advanced:

- It supports more disciplined workflows and stronger operational control.
- It fits agencies and power users better than the public Free surface.

### 6. Brand adaptation, critique, and upgrade layer

- deeper recommendation tracks beyond the public Free baseline
- extract design tokens
- plan brand adaptation
- plan rebuild strategy
- critique rebuild strategy
- critique Elementor output
- stronger scenario-based upgrade guidance
- richer stack-shaping logic
- deeper skill-profile and addon-profile use when justified

Why this belongs in Advanced:

- It deepens contextualization, planning quality, critique quality, and confidence.
- It is a clear productivity and workflow upgrade.

### 7. Elementify Library as curated premium layer

Advanced should visibly introduce the `Elementify Library`.

This means:

- curated premium collections across conversion, service-pages, trust-components, and theme-builder families
- browse the curated premium catalog
- inspect premium assets before use
- plan the import locally into the current Elementor Library
- execute the local import through the dedicated library import seam
- curated premium templates
- stronger reusable structures
- premium starters and assets
- local-site operational use through or alongside the local Elementor Library

This does not mean:

- cloud library
- cross-site sync
- Studio-style storage or delivery logic

## Advanced Workflow Set

### Advanced workflow 1: Scenario-first routing

Route deep relaunch, migration, premium rollout, and critique-first work into the right Advanced execution path.

### Advanced workflow 2: Deep bootstrap and stack shaping

Move from a valid baseline toward a more powerful, better-shaped stack when the scenario genuinely benefits.

### Advanced workflow 3: Theme Builder delivery

Move from page-level composition to site-wide structural delivery.

### Advanced workflow 4: Deeper creator and media flows

Use richer creator support to get better results faster.

### Advanced workflow 5: Governed change workflows

Let AI or operators prepare changes while preserving a stronger review layer.

### Advanced workflow 6: Brand adaptation and critique

Use deeper planning and critique to align output quality, not just create output.

The Advanced critique layer should not stop at advice:

- it should validate the target payload
- turn critique into explicit repair steps
- and, when needed, prepare a governed follow-up change entry instead of leaving the operator with only abstract guidance

### Advanced workflow 7: Premium library usage

Use curated Elementify assets through a `list -> inspect -> plan -> import locally` flow as a stronger reusable building layer on the current site.

### Advanced workflow 8: Upgrade and stack shaping

Use scenario-aware runtime guidance to distinguish:

- keep current stack
- Elementor Pro for builder / structural gaps
- Elementify Advanced for workflow depth
- one curated supported pro addon profile when it is genuinely justified

### Advanced workflow 9: Variant, rollout, and repair depth

Use stronger skill profiles, deeper creator flows, and richer repair loops to reduce work on larger or more ambitious site outcomes.

The productivity layer should be explicit in runtime output:

- variant moves
- reuse-light moves
- follow-up mode
- time-saving moves

## What Advanced Explicitly Does Not Include

- Studio cloud library
- cross-site template storage
- cross-site sync
- delivery orchestration as a live product promise
- team and department cloud semantics
- marketplace launch behavior

## Product Guardrails

- Advanced remains private in the Forgejo primary repo.
- Advanced must remain separable from the public Free mirror.
- Advanced extends the Free core, but Free must not depend on Advanced modules.
- Advanced may brand the premium layer as `Elementify Library`, but must not imply Studio cloud behavior.
- Advanced may recommend richer addon profiles and deeper skill profiles, but must remain scenario-led and not turn into stack inflation.

## Strategic Interpretation

Advanced is not “more buttons”.

It is:

- the deeper workflow layer
- the stronger creation layer
- the better critique and guidance layer
- the first premium library layer
- the richer stack-shaping and upgrade-guidance layer

That is why it should be framed as a productivity upgrade rather than as a feature bundle.
